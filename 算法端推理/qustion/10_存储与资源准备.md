本章解释：系统如何同时支持 `minio://` 与本地路径；以及任务执行前如何把输入资源准备成“最终可读的本地文件路径（resolved_source_url）”。

### 1.1 URL 类型判断
- `URLHandler.is_minio_url(url)`：是否以 `minio://` 开头

## 2. StorageService：统一下载/上传/复制/删除

- [shared/storage_service.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/shared/storage_service.py#L16-L250)

### 2.1 download()：MinIO → fget_object，本地 → copy_file
核心逻辑：
- 若解析为 MinIO：使用 MinioClient 下载到 local_path
- 若解析为 file：用 FileSystemHandler 复制到 local_path

- MinIO → MinIO（服务器端 copy）
- MinIO → 本地（下载）
- 本地 → MinIO（上传）
- 本地 → 本地（文件复制）

位置：
- [storage_service.py:copy](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/shared/storage_service.py#L155-L208)

## 3. ResourcePreparer：任务执行前的资源准备器
实现：
- [scanner/utils/resource_preparer.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/utils/resource_preparer.py#L11-L131)

### 3.1 prepare(source_url, task_id) 的返回值契约
返回一个二元组：
1. `resolved_source_url`：最终用于推理的本地文件路径
2. `predownloaded_files`：预下载产生的临时文件列表（用于后续清理）

代码位置：
- [resource_preparer.py:prepare](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/utils/resource_preparer.py#L36-L89)

### 3.2 MinIO URL 的处理
当 `source_url` 是 `minio://...`：
1. 生成临时文件名（按任务 ID 带前缀）
2. `StorageService.download(source_url, temp_file_path)`
3. 返回 temp_file_path 作为 resolved_source_url，并记入 predownloaded_files

关键代码：
- [resource_preparer.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/utils/resource_preparer.py#L54-L66)
- [resource_preparer.py:_predownload_file](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/utils/resource_preparer.py#L91-L131)

### 3.3 非 MinIO URL 的处理（可配置）
当 `predownload_non_minio=True`：
- 也会走同样的预下载逻辑（本地路径会被复制到临时目录）

关键代码：
- [resource_preparer.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/utils/resource_preparer.py#L67-L88)

## 5. 一个端到端示例（概念追踪）
假设数据库里某条任务的 `source_url = "minio://bucket/path/a.svs"`：
1. TaskPreprocessor 调用 `ResourcePreparer.prepare(source_url, task_id)`  
   - [preprocessor.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/preprocessor.py#L196-L213)
2. URLHandler 解析为 MinIO target  
   - [url_handler.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/shared/url_handler.py#L105-L159)
3. StorageService.download 分发到 MinioClient.fget_object  
   - [storage_service.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/shared/storage_service.py#L51-L80)
4. 返回 `resolved_source_url=/tmp/task_123_xxx.svs`  
5. TaskFactory 创建 Task，把 resolved_source_url 放入 task.resolved_source_url  
   - [factory.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/scanner/factory.py#L92-L97)
6. 后续推理读取的入口永远是 `task.source_url`（会优先返回 resolved_source_url）  
   - [task.py:source_url](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/task.py#L87-L91)
