# 01 启动与生命周期（app 模块）
本章解释服务如何从命令行启动，并进入“长期运行（轮询/监听）”的主循环；以及停止时如何优雅关闭。

## 1. 启动入口：`python -m app` / `app.main.main()`
### 1.1 `app/__main__.py`
当你执行 `python -m app`，Python 会执行该文件：
- [app/__main__.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/__main__.py#L1-L6)

关键点：它只做一件事：调用 `main()`。

### 1.2 `app/main.py`
- [app/main.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/main.py#L11-L21)

这里的 `main()` 有两个关键动作：
1. 设置多进程启动方式为 `spawn`
   - 目的：避免 GPU/CUDA 与 `fork` 在多进程下的隐患；并让子进程更可控。
2. 进入 CLI：`cli()`

> 你可以把 `app/main.py` 理解为“进程级入口”：它决定多进程行为，并将控制权交给 CLI 子命令。

## 2. CLI 命令：`start/config/health`
CLI 在：
- [app/cli_commands.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/cli_commands.py#L17-L68)

你通常会用：

`start` 子命令的核心两行：
- 创建应用对象：`app = Image_cellInferApp()`
- 启动：`asyncio.run(app.start())`

代码位置：
- [cli_commands.py:start](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/cli_commands.py#L26-L36)

这意味着：**应用的生命周期由 `Image_cellInferApp.start()` 这个协程驱动**。

### 3.1 `start()`：启动顺序与“启动失败的回滚点”
入口：
- [application.py:start](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L98-L137)

启动顺序（按代码顺序）：
1. 初始化日志与信号处理（在 `__init__`）
   - 日志初始化：`setup_logging(self.config.logging)`
   - 信号：`register_signal_handlers(logger, self._shutdown_event)`
   - 位置：[application.py:__init__](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L40-L57)、[signals.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/signals.py#L12-L27)
2. 注册 DB 监听器工厂
   - `register_builtin_listeners()`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L107-L111)、[db/listener/factories.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/db/listener/factories.py#L106-L120)
3. 清理临时目录（避免上次残留）
   - `TempFileManager.cleanup_temp_directory(self.config.image_cell.temp_dir)`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L107-L111)
4. 初始化数据库连接池
   - `await db_manager.initialize()`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L107-L111)、[db/connection.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/db/connection.py#L28-L71)
5. 健康检查（决定能否继续启动）
   - `health = await self.health_check()`
   - 若关键组件 unhealthy：立刻 `await db_manager.close()` 并抛出 `StartupError`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L113-L122)、[health_service.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/health_service.py#L19-L75)
6. 创建编排器
   - `self._factory.create_image_cell_orchestrator()`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L124-L129)、[orchestrator_factory.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/orchestrator_factory.py#L32-L46)
7. 并行启动服务（实际就是启动编排器 + 可选内存监控）
   - `_startup_services()`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L159-L179)
8. 进入主循环
   - `while self._running and not self._shutdown_event.is_set(): await asyncio.sleep(1)`
   - 位置：[application.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L131-L137)

你可以把第 5 步看作一个“启动门槛”：不满足就终止并清理数据库连接池。

### 3.2 `stop()`：停止顺序
入口：
- [application.py:stop](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L139-L156)

停止顺序：
1. 设置 `_running=False`，让主循环自然退出
2. `_shutdown_services()`：停止编排器、停止内存监控
3. `db_manager.close()`：关闭数据库连接池
4. `cleanup_logging()`：清理日志系统资源

编排器停止使用了“排空停止”语义：
- [application.py:_stop_orchestrator](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/application.py#L207-L217)
- `await orchestrator.stop_after_current(timeout=600.0)`

这意味着：应用会尽量等正在处理中的任务完成（或超时），再整体退出。

## 5. 优雅关闭：信号触发 `_shutdown_event`
信号处理：
- [app/signals.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/app/signals.py#L12-L27)

当收到 `SIGINT/SIGTERM`：
- `shutdown_event.set()` 被触发
- `application.py` 主循环检测到后会退出并进入 `finally: await self.stop()`
