# 02 配置系统（config 模块）
本章解释配置从哪里来、如何映射环境变量、以及哪些配置会直接影响任务扫描与推理流水线。

## 1. 配置入口：`AppConfig(BaseSettings)`
核心文件：
- [config/app_settings.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L22-L259)

`AppConfig` 继承自 `pydantic_settings.BaseSettings`，意味着：
- 配置字段既可以由默认值提供，也可以由环境变量覆盖
- 字段常通过 `alias='ENV_NAME'` 显式绑定环境变量名

示例：
- `db_host` 别名为 `DB_HOST`
- `db_port` 别名为 `POSTGRES_EXTERNAL_PORT`
- `image_cell_read_workers` 别名为 `Image_cell_READ_WORKERS`

这些映射在同一文件中集中定义，属于“单一真相源”。

## 2. 配置装配：把“扁平 env 字段”组装成“结构化配置对象”
`AppConfig` 并不是直接把 `db_host/db_port/...` 暴露给业务层；而是通过属性组装为强类型子配置：
- `database -> DatabaseConfig`
- `minio -> MinIOConfig`
- `logging -> LoggingConfig`
- `image_cell -> Image_cellConfig`
- `orchestrator -> OrchestratorConfig`

代码位置：
- [app_settings.py:database/minio/logging/image_cell/orchestrator](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L288-L364)

这套做法的好处：
- 对业务层：只接触结构化对象（例如 `config.image_cell.read_workers`），语义清晰
- 对运行时：仍可用环境变量覆盖（例如容器里注入 `Image_cell_SHM_POOL_SIZE=64`）

## 3. 运行环境自动适配（host 智能判断）
`db_host` 与 `minio_host` 会根据是否在容器内运行做修正：
- 数据库 host：优先用显式 `DB_HOST`，否则容器内默认 `postgres`，本地默认 `localhost`
- MinIO host：容器内默认 `minio`

代码位置：
- [app_settings.py:_get_database_host](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L260-L278)
- [app_settings.py:_get_minio_host](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L280-L287)

你可以把这理解为：同一份镜像/代码，部署到本地或 Docker Compose 时，不需要改配置文件，只需要靠环境变量/容器检测即可切换。

## 4. `get_config()`：线程安全单例
`get_config()` 使用双重检查锁定，在多线程环境下确保只创建一次 `AppConfig`：
- [app_settings.py:get_config/reload_config](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L367-L391)

## 6. 影响推理流水线性能与资源的关键配置
### 6.1 共享内存与队列（决定吞吐与背压）
字段：
- `image_cell_shm_pool_size`（共享内存块数）
- `image_cell_queue_size`（流水线队列容量）

定义位置：
- [app_settings.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L101-L181)

使用位置：
- 共享内存池初始化： [core/services/builder.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/core/services/builder.py#L160-L171)
- 各组件输入队列容量： [core/services/builder.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/core/services/builder.py#L213-L277)

直觉：
- `shm_pool_size` 太小会导致 TileReader 等待共享内存可用，吞吐下降
- `queue_size` 太小会让上游更早阻塞（更强背压），减少内存占用但吞吐可能下降

### 6.2 Reader/Filter/Normalizer/Inferencer 并发
字段定义：
- [app_settings.py](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/config/app_settings.py#L109-L174)

使用位置：
- [builder.py:_create_components](file:///home/zyh/NewMedLabel/medlabel_image_cell_inference/src/image_cell_processing/core/services/builder.py#L216-L289)

并发模型（概念上）：
- Reader：线程池（多线程 IO）
- Filter：进程池（CPU 多进程）
- Normalizer：进程池（CPU 多进程）
- Inferencer：多进程（可能使用 GPU/CPU 混合设备）
- DBWriter：异步并发写入（Semaphore 控制并发）
